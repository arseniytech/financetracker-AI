Трекер расходов
1 Приложение для учёта личных расходов. Аудитория, люди которые следя за своими деньгами.
Сценарии использования:

добавить расход
посмотреть историю
отфильтровать по категории
удалить запись

2 Функциональные требования
описание основных функций системы:

добавление расхода (сумма, категория, дата, описание)
просмотр списка всех расходов
фильтрация по категории
удаление расхода
показ общей суммы

CRUD:

Create - POST /expenses
Read - GET /expenses
Update -
Delete - DELETE /expenses/:id

3 Архитектура
Клиент-серверная. Фронт на React шлёт запросы на бэк через HTTP API. Бэк обрабатывает и сохраняет в SQLite.
React -> HTTP -> express -> SQLite

4 Описание API

Вид API — REST.

GET /expenses - получить все расходы
POST /expenses - добавить расход
DELETE /expenses/:id - удалить расход по id
POST /auth/register - регистрация пользователя
POST /auth/login - вход

Аутентификация базовая — email + пароль, хранится в БД.

5 Требования к интерфейсу

форма добавления расхода
таблица с историей
фильтры по категориям
боковое меню
